# 类型规范

## 1. TypeScript（admin / client / h5）

### 1.1 基本约束

- 全量开启 `strict`（已在各 `tsconfig.json` 配置），**禁止关闭**。
- **禁止 `any`**。确实未知用 `unknown` 再收窄；第三方无类型时本地补 `d.ts`。
- 禁止 `@ts-ignore` 掩盖错误；必须临时绕过时用 `@ts-expect-error` 并注明原因与清理条件。
- 非空断言 `!` 仅在你能证明非空时使用，优先用可选链 `?.` 与默认值。

### 1.2 类型 vs 接口

- 对象结构优先 `interface`（可声明合并、报错更友好）；联合/交叉/工具类型用 `type`。
- 组件 Props / Emits 用 `defineProps<T>()` / `defineEmits<T>()` 的**类型化**写法，不用运行时数组写法。

### 1.3 接口出入参类型

- `api/` 下每个请求都要有**入参类型**和**返回类型**，返回统一遵循后端契约：

```ts
// 与后端 Result<T> 对齐：code === 0 为成功
export interface ApiResult<T> {
  code: number;
  message: string;
  data: T;
}
```

- 列表分页统一类型（参考 admin 现有 `ResultTable`）：`{ list: T[]; total: number; pageSize: number; currentPage: number }`。
- 金额/大整数（订单号、ID）若后端用 `Long`/`BigDecimal`，前端**用 `string` 承接**，避免 JS `number` 精度丢失（见[数据库规范](./04-数据库规范.md)）。

### 1.4 枚举与常量

- 用 `as const` 对象 + 派生类型，少用 TS `enum`（避免运行时产物与 tree-shaking 问题）：

```ts
export const OrderStatus = { Pending: 0, Filled: 1, Canceled: 2 } as const;
export type OrderStatus = (typeof OrderStatus)[keyof typeof OrderStatus];
```

### 1.5 环境变量类型

- 新增 `VITE_*` 变量后必须同步类型：admin 在 `types/global.d.ts` 的 `ViteEnv` 接口 + `build/utils.ts` 默认值 + `.env.*`，**三处缺一会编译失败**。

## 2. Java（server）

### 2.1 出入参类型

- 接口出入参一律用 **DTO**，禁止直接暴露 `entity`。
- 返回值统一 `Result<T>`，`T` 用明确类型；演示期可用 `Map`，**正式接口必须定义具名 VO/DTO**。
- 入参校验用 Jakarta Validation 注解（`@NotBlank`/`@NotNull`/`@Min` 等）+ controller 上 `@Valid`，由 `GlobalExceptionHandler` 统一兜底。

### 2.2 金额与数值（外汇核心）

- 金额、汇率、手续费一律 **`BigDecimal`**，构造用字符串 `new BigDecimal("1.23")`，**禁止用 `double` 构造**。
- 比较用 `compareTo`，不用 `equals`；除法必须指定 `scale` 与 `RoundingMode`。
- 数据库大整数主键/订单号用 `Long`，序列化给前端时**转 `String`**（Jackson 全局配置 Long→String）。

### 2.3 空值与 Optional

- 可能为空的返回用 `Optional<T>`，不返回 `null` 集合（返回空集合）。
- 入参非空由校验注解保证；内部方法对外部不可信值做防御性判空。

### 2.4 时间与时区

- 用 `java.time`（`LocalDateTime`/`Instant`），**禁止** `java.util.Date`。
- 对外时间统一 UTC 或带时区，前后端约定一致的格式（ISO-8601）。

### 2.5 不可变与封装

- DTO/VO 字段尽量不可变（构造或 builder 注入），对外只读。
- 常量集中在常量类或枚举，避免魔法值散落。
