# 安全规范

> 交易类平台对安全要求极高，本章为强制要求。

## 1. 密钥与敏感信息（最高优先级）

- **任何密钥/令牌/密码/私钥严禁出现在源码、提交历史、日志、前端打包产物中。**
- 配置经环境变量或密钥管理服务注入；本地用根目录 `.env`（已被 `/.env` 规则忽略，不得改动该忽略）。
- 一旦密钥泄露（误提交、贴进聊天/工单等）：**立即 Revoke 并轮换**，再处理代码。
- 提交前自查：`git diff --cached` 不含 `ghp_`、`sk-`、`password=`、`PRIVATE KEY` 等特征串。
- 前端 `import.meta.env` 只能放**公开**配置；任何 `VITE_*` 都会进入打包产物，**不得放密钥**。

## 2. 认证与授权

- 当前演示用内存账号（`AuthController`），**正式必须替换**为：密码 `BCrypt` 加盐哈希存储 + JWT/会话鉴权。
- Token：accessToken 短时效 + refreshToken 滚动刷新；前端已具备刷新机制（`refresh-token`，白名单见 `admin/src/utils/http`）。
- **接口级 + 数据级双重鉴权**：不仅校验“是否登录”，还要校验“该用户是否有权访问该条数据”（防越权 IDOR）。
- 后端做**强制**权限校验，前端的按钮/菜单权限只是体验，**不能作为安全边界**。
- 遵循最小权限原则；管理端高危操作（资金、风控、用户）需二次确认/审计。

## 3. 输入校验与注入防护

- **所有外部输入不可信**：参数、Header、文件名、回调都要校验。
- 后端用校验注解 + 业务校验双层；前端校验仅为体验。
- SQL：统一参数化（MyBatis `#{}`、JPA 参数绑定），**禁止字符串拼接 SQL**，禁止 `${}` 拼接用户输入。
- 命令/路径：禁止用用户输入拼接系统命令或文件路径（防命令注入/路径穿越）。

## 4. XSS / CSRF / 点击劫持

- 前端默认信任 Vue 的转义；**禁止滥用 `v-html`**，必须用时先做白名单消毒。
- 富文本/外链内容服务端消毒后再下发。
- 写操作使用 Token 鉴权（非纯 Cookie 会话可天然抗 CSRF）；如用 Cookie 会话需加 CSRF Token 与 `SameSite`。
- 响应头建议：`X-Content-Type-Options: nosniff`、`X-Frame-Options/CSP frame-ancestors`、`Content-Security-Policy`。

## 5. 传输与跨域

- 生产**全站 HTTPS**，敏感接口禁止明文。
- CORS：`config/CorsConfig` 当前为开发期放开（`allowedOriginPatterns("*")`）。**上线必须收敛为白名单域名**，不可 `*` + 携带凭证长期共存。

## 6. 越权与业务风控（交易特有）

- 下单/出金/改密等关键操作：校验资金、状态机合法性、幂等（防重复提交）、限频。
- 金额、汇率等关键参数**以服务端为准**，绝不信任前端传入的价格/金额。
- 关键操作记录**审计日志**（操作人、时间、IP、前后值），不可篡改。

## 7. 敏感数据处理

- 落库前对密码（哈希）、关键证件/银行卡（加密，见 [API 加密规范](./05-API与加密规范.md)）处理。
- 日志、响应中对手机号、证件、卡号做**脱敏**（掩码），禁止打印完整敏感字段与原始报文。
- 异常信息不向前端暴露堆栈/SQL/内部细节，统一走 `GlobalExceptionHandler` 返回友好提示。

## 8. 依赖与配置安全

- 定期升级依赖，关注 CVE；不引入来路不明的包。
- 关闭无关的执行器/调试端点；生产 `application.yml` 不暴露敏感配置。
- 错误码不泄露系统内部实现（如具体框架版本、路径）。
