# UI 规范

## 0. 总则（强制）

- **UI 一律按产品/用户需求实现**：不臆造、不自由发挥、不擅自增删需求外的页面与控件；需求不明确先确认，再开发。
- **组件复用优先，禁止重复创建**：实现任何 UI 前，先在本端组件库中查找可复用项——
  1. admin 先查 `src/components`（26 个 `Re*` 组件）与 Element Plus；
  2. client/h5 先查 Vant 与本端 `src/components`。
  - **确实没有**可用组件时，才新建；新建组件须做成**可复用**（props/slots/events 通用化），放对位置（通用 → `components/`，业务 → `views/**/components`）。
  - 严禁复制粘贴产生功能重复的组件；发现重复应合并复用。

## 1. 组件库选型（按端固定）

| 端 | 组件库 | 说明 |
| --- | --- | --- |
| `admin/` | **Element Plus**（pure-admin 生态）+ `src/components` 下 26 个 `Re*` 复用组件 | PC 管理后台 |
| `client/` `h5/` | **Vant**（移动端，自动按需导入） | 移动端 H5 |

- **先复用再造轮子**：admin 优先用 `src/components`（`ReDialog`、`ReDrawer`、`RePureTableBar`、`ReSegmented` 等）与 Element Plus；移动端优先用 Vant。
- 同一端不混用多套 UI 库；确需引入新组件先评审。

## 2. 复用与拆分

- 重复出现 ≥ 2 次的 UI 片段抽成组件；单组件超 500 行按[代码规范](./01-代码规范.md)拆分。
- 业务组件（带业务语义）放 `views/**/components`，通用组件放 `components/`。
- 组件对外只暴露必要 props/events/slots，内部实现可替换，保持稳定接口。

## 3. 主题与样式

- **统一用设计变量/Design Token**，禁止硬编码颜色、字号、间距：
  - admin：用 Element Plus CSS 变量（`--el-color-primary`、`--el-text-color-*` 等）与 pure-admin 主题机制；切换主题靠变量，不写死色值。
  - 移动端：用 Vant 主题变量（`--van-*`）。
- 间距/圆角/字号走规范刻度（如 4 的倍数），不随手取值。
- 暗黑模式：颜色一律走变量，确保暗色主题自动适配（admin 已内置 DarkMode）。
- 样式默认 `scoped`，穿透用 `:deep()`；禁止全局选择器污染。

## 4. 移动端适配（client / h5）

- `viewport` 已在 `index.html` 配置（`width=device-width, initial-scale=1, viewport-fit=cover`）。
- 优先弹性布局（flex）+ 百分比/`vw`；需要 rem 适配时引入统一方案（如 `postcss-px-to-viewport`/`amfe-flexible`），全项目一致，不混用。
- 触控目标 ≥ 44×44px；考虑刘海屏安全区（`env(safe-area-inset-*)`）。
- 列表/行情等长列表用虚拟滚动或分页，避免一次渲染海量 DOM。

## 5. 交互一致性

- 加载、空状态、错误状态**三态必做**：用 Vant/Element 的 loading、empty、错误提示，禁止白屏无反馈。
- 反馈统一：成功/失败提示用各端统一封装（admin 用 `@/utils/message`，移动端用 Vant `showToast/showDialog`），文案风格一致。
- 高危操作（删除、出金、平仓）必须二次确认。
- 表单：即时校验 + 提交前整体校验；提交中禁用按钮防重复提交。
- 金额展示按品种精度格式化（千分位、固定小数位），涨跌用统一红绿语义（注意区分地区习惯并统一约定）。

## 6. 可访问性与国际化

- 图片 `alt`、图标按钮 `aria-label`/`title`；可键盘操作。
- 文案走 i18n（admin 已有 `locales/`），**禁止**散落硬编码中文/英文字符串在模板里（除占位/演示）。
- 时间、数字、货币按 locale 格式化。

## 7. 性能

- 路由级懒加载（`() => import()`，三端均已采用）。
- 图片用合适尺寸与格式，移动端注意首屏体积；按需引入组件与图标。
- 避免在模板里放重计算，用 `computed` 缓存；长列表避免不必要的响应式深层。
