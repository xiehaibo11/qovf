# QOVF 开发规范

本目录是 QOVF 平台的开发规范，**所有提交代码均需遵守**。文档按主题拆分管理，单篇控制在 300–500 行以内，超出即继续拆分。

## 适用范围

| 子项目 | 技术栈 | 主要规范 |
| --- | --- | --- |
| `admin/` | Vue 3 + Vite + Pinia + TS + Element Plus（基于 vue-pure-admin） | 代码 / 类型 / UI / 状态管理 |
| `client/` `h5/` | Vue 3 + Vite + Pinia + TS + Vant（移动端） | 代码 / 类型 / UI / 状态管理 |
| `server/` | Spring Boot 3 + Java 17 | 代码 / 类型 / 数据库 / API 加密 / 安全 |

## 核心红线（违反即打回）

- **金额一律高精度**：Java 用 `BigDecimal`、DB 用 `DECIMAL`，**严禁** `float`/`double`/`Number` 参与金额计算。详见[数据库规范](./04-数据库规范.md)。
- **统一返回契约不可破坏**：后端必须返回 `Result<T>`（`code=0` 成功），前端以 `code === 0` 判断成功。详见 [CLAUDE.md](../CLAUDE.md) 与 [API 规范](./05-API与加密规范.md)。
- **密钥/令牌严禁入库入仓**：任何 token、密码、私钥、密钥不得出现在源码、提交、日志中；仅通过环境变量/密钥管理注入。详见[安全规范](./03-安全规范.md)。
- **单文件超 500 行必须拆分**：构建时由 `scripts/check-file-lines.mjs` 强制校验（前端 `pnpm build`、后端 `mvn` 的 `validate` 阶段均会运行，超限即失败）。见[代码规范](./01-代码规范.md)。
- **禁止写模拟/假数据**：业务代码不得内置硬编码假数据、随机造数、mock 返回。数据一律来自真实接口/数据库。见[代码规范](./01-代码规范.md#9-禁止模拟数据)。
- **UI 按需求与复用优先**：UI 一律按产品需求实现，组件**先复用后创建**，禁止重复造组件；确无可用组件才新建。见[UI 规范](./06-UI规范.md)。
